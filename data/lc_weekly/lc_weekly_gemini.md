### 1. The "Graph & Tree Mastery" Set

**Biweekly Contest 102**
This contest is a classic because it mirrors the Google interview's tendency to ask about level-order traversal and shortest paths in non-traditional grids.

* **Key Focus:** BFS/DFS, Tree level-order construction, and Dijkstra-like logic.
* **Why it's good:** The "Design Graph with Shortest Path Calculator" problem is a quintessential Google-style "system design meets algorithms" task.

### 2. The "Optimization & Greedy" Set

**Biweekly Contest 103**
This set focuses on how to handle arrays and ranges efficiently, which is a core Google competency.

* **Key Focus:** Maximize/Minimize values using Priority Queues and Heap-based Greedy strategies.
* **Why it's good:** It tests your ability to optimize from $O(n^2)$ to $O(n \log n)$, a frequent requirement in the second half of a 45-minute interview.

### 3. The "Binary Search on Answer" Set

**Weekly Contest 331**
Google is famous for problems where the solution isn't a direct calculation but a binary search over a range of possible results.

* **Key Focus:** Binary search on answer, Heap operations, and Prefix Sums.
* **Why it's good:** The problem "Minimizing the Maximum Ability" (or similar variations) is a common pattern used to filter Senior SWE candidates.

### 4. The "Classic DP & String" Set

**Weekly Contest 341**
This contest covers nested logic and string manipulation, which tests your ability to handle edge cases without getting lost in your own code.

* **Key Focus:** Dynamic Programming, String state machines, and Stack-based parsing.
* **Why it's good:** Google often asks "Minimum additions/deletions to make a string valid," and this contest provides excellent practice for that logic.
